상세 설계 ②③④ — Ingestion → Stream → Storage

API Gateway에서 S3 Data Lake까지의 서버리스 파이프라인

② Ingestion — API Gateway (HTTP API)

엔드포인트 설계

POST /v1/events          ← 배치 이벤트 수신
POST /v1/events/beacon   ← sendBeacon (페이지 이탈)
GET  /v1/config          ← SDK 원격 설정 (샘플링률 등)
GET  /v1/health          ← 헬스체크
왜 HTTP API인가?
  • REST API 대비 70% 저렴 ($1.00 vs $3.50 / 백만 요청)
  • 지연 시간 더 낮음 (~10ms vs ~30ms)
  • CORS, 스테이지 지원

인증 & 보안

API Key 인증:

  • 앱별 API Key 발급 (web-prod, ios-prod 등)
  • Usage Plan으로 Rate Limit 설정
  • Key는 공개 클라이언트에 포함 (비밀이 아님)

남용 방지:

  • Rate Limit: 1,000 req/s per key
  • Payload 크기 제한: 512KB
  • WAF: Bot 탐지, IP 기반 제한
  • Origin 검증 (Web — Referer/Origin 헤더)

Lambda Transform 상세

런타임: Python 3.12
메모리: 256MB
타임아웃: 60초

GeoIP Enrichment 방식:
  • MaxMind GeoLite2 DB를 Lambda Layer로 패키징
  • IP → Country/City/Region 변환 후 IP 폐기
  • DB 월 1회 자동 업데이트 (EventBridge + Lambda)

💡 핵심 최적화: API Gateway → Firehose 직접 연결

API Gateway의 Service Integration을 사용하면 수집 Lambda 없이 Firehose에 직접 전송 가능합니다. 이벤트 수신에 Lambda를 쓰지 않으므로 비용 절감 + 지연 감소 + 콜드 스타트 제거. Transform Lambda는 Firehose 내부에서만 호출됩니다.

④ Storage — S3 Data Lake

버킷 구조 & 파티셔닝

s3://rum-data-lake/
├── raw/                          ← Firehose 출력
│   └── platform=web/
│       └── year=2026/
│           └── month=04/
│               └── day=01/
│                   └── hour=09/
│                       └── events-001.parquet
├── aggregated/                   ← 집계 Lambda 출력
│   ├── hourly/
│   │   └── metric=cwv/
│   │       └── 2026-04-01-09.parquet
│   └── daily/
│       └── metric=user_journey/
│           └── 2026-04-01.parquet
├── errors/                       ← Firehose 실패 레코드
└── config/                       ← SDK 원격 설정 JSON

S3 Lifecycle Policy

raw/ 0~90일: S3 Standard
90일 후: 삭제
aggregated/ 0~90일: S3 Standard
90~365일: S3 IA
365일 후: Glacier
errors/ 30일 후: 삭제

예상 용량 (DAU 5만)

일 이벤트: ~500만 건
Parquet 일 용량: ~500MB
월 용량: ~15GB
90일 raw: ~45GB
(JSON 대비 ~80% 압축)

Glue Data Catalog 테이블

rum_events (raw)
  • Firehose 파티션 자동 등록
  • Athena에서 바로 쿼리 가능
rum_hourly_metrics (aggregated)
  • 시간별 집계 메트릭
  • 대시보드 주 데이터 소스
rum_daily_summary (aggregated)
  • 일별 요약 (유저, 에러, 성능)
  • 트렌드 분석용
