상세 설계 ① — SDK Layer (클라이언트)

Web과 Mobile에서 이벤트를 수집하여 백엔드로 전송하는 경량 SDK

Web SDK

수집 이벤트

Performance LCP, FID/INP, CLS, TTFB, FCP
Navigation 페이지뷰, SPA 라우트 변경, 히스토리 변경
User Action 클릭, 스크롤 깊이, 폼 인터랙션
Error JS 에러, 미처리 프로미스, 콘솔 에러
Resource XHR/Fetch 요청, 응답 시간, 상태 코드

구현 전략

방식: 자체 경량 SDK (TypeScript)

크기 목표: gzip 후 <10KB

핵심 API 활용:

  • PerformanceObserver — CWV 수집
  • window.onerror / onunhandledrejection
  • MutationObserver — SPA 라우트 감지
  • navigator.sendBeacon — 페이지 이탈 시 전송

Mobile SDK

수집 이벤트

Performance 앱 시작 시간, 화면 로딩, 프레임 드랍
Screen 화면 진입/이탈, 체류 시간
User Action 탭, 스와이프, 스크롤
Crash 미처리 예외, ANR (Android), OOM
Network API 호출, 응답 시간, 에러율

구현 전략

방식: 프레임워크별 SDK 래퍼

  • React Native: JS 레이어 공유 + Native 모듈
  • Flutter: Dart 플러그인 + MethodChannel
  • Native: Swift/Kotlin 각각 구현

핵심: Web SDK와 동일한 이벤트 스키마 공유

공통: 이벤트 스키마 & 전송

통합 이벤트 스키마

{
  "session_id": "uuid",
  "user_id": "hashed_id | anonymous",
  "device_id": "uuid",
  "timestamp": 1712000000000,
  "platform": "web | ios | android",
  "app_version": "2.1.0",
  "event_type": "performance | action | error | navigation | resource",
  "event_name": "lcp | click | js_error | page_view | xhr",
  "payload": { /* 이벤트별 데이터 */ },
  "context": {
    "url": "/products/123",
    "screen_name": "ProductDetail",
    "device": { "os": "iOS 17", "model": "iPhone 15", "browser": "Chrome 120" },
    "geo": { /* 서버사이드에서 IP 기반 enrichment */ },
    "connection": { "type": "4g", "rtt": 50 }
  }
}

전송 전략

배치 전송:

  • 이벤트를 로컬 버퍼에 쌓음
  • 30초 간격 또는 10개 누적 시 전송
  • gzip 압축 (70~80% 절감)

안정성:

  • 오프라인 시 IndexedDB/SQLite 큐잉
  • 재시도 (exponential backoff)
  • 페이지 이탈: sendBeacon
  • 앱 백그라운드: flush on pause

🔒 프라이버시 & 샘플링

PII 보호:
  • 이메일/전화번호 자동 마스킹
  • 입력 필드 값 수집 안 함
  • user_id는 해시 처리
  • IP 주소 저장 안 함 (GeoIP 후 폐기)
샘플링:
  • 기본 100% (중규모에서 감당 가능)
  • 트래픽 증가 시 세션 기반 샘플링
  • 에러 이벤트는 항상 100% 수집
  • 서버사이드 config로 원격 제어
