AWS RUM 아키텍처 접근 방식

3가지 접근 방식을 비교해보세요. 클릭하여 선호하는 방식을 선택할 수 있습니다.

A

🏗️ Custom Pipeline (Kinesis 중심) — 추천

구성: SDK → API Gateway → Kinesis Data Streams → Lambda (변환/강화) → OpenSearch + S3

대시보드: Grafana

장점

  • 가장 비용 효율적 (이벤트당 비용 최소)
  • Web + Mobile 동일 파이프라인
  • 기존 OpenSearch 클러스터 활용
  • 완전한 커스터마이징 가능
  • 세션 리플레이 확장 용이

단점

  • 초기 개발 공수 가장 높음 (4~6주)
  • SDK 직접 구현 또는 오픈소스 채택 필요
  • 운영 부담 존재

예상 월 비용 (DAU 5만 기준): $200~500/월
Kinesis ~$80 + Lambda ~$30 + S3 ~$20 + OpenSearch 기존 활용 + Grafana ~$50

B

☁️ CloudWatch RUM + Custom Mobile

구성: Web: CloudWatch RUM (관리형) / Mobile: Custom → Kinesis → OpenSearch

대시보드: CloudWatch Console (Web) + Grafana (Mobile)

장점

  • Web 빠르게 설정 가능 (1~2일)
  • AWS 관리형 서비스 (Web 부분)
  • CloudWatch 네이티브 통합

단점

  • CloudWatch RUM은 Web 전용 (모바일 미지원)
  • 두 시스템 운영 필요 → 복잡도 증가
  • 대시보드 통합 어려움 (Web/Mobile 분리)
  • CloudWatch RUM 가격이 Datadog 대비 크게 절감되지 않음
  • 커스터마이징 제한적

예상 월 비용 (DAU 5만 기준): $500~1,200/월
CloudWatch RUM ~$500 (10만 세션 × $1/1000) + Mobile 파이프라인 ~$200 + Grafana ~$50

C

🔭 OpenTelemetry 기반 (OTel + Grafana)

구성: OTel SDK (Web/Mobile) → OTel Collector (ECS) → S3 + OpenSearch

대시보드: Grafana

장점

  • 벤더 중립 — 오픈소스 표준
  • 커뮤니티 활발, 에코시스템 풍부
  • 백엔드 트레이싱(X-Ray)과 자연스럽게 연결
  • Web + Mobile 단일 표준

단점

  • OTel RUM SDK가 아직 성숙 단계 (특히 모바일)
  • OTel Collector 클러스터 운영 필요
  • 세션 리플레이 네이티브 미지원
  • 초기 설정 복잡도 높음

예상 월 비용 (DAU 5만 기준): $300~700/월
ECS (Collector) ~$150 + S3 ~$20 + OpenSearch 기존 활용 + Grafana ~$50

💡 추천: A — Custom Pipeline

비용 절감이 핵심 목표이고, 기존 OpenSearch를 활용할 수 있으며, Web/Mobile을 하나의 파이프라인으로 통합할 수 있습니다. 초기 개발 공수가 높지만 장기적으로 가장 경제적이고 확장 가능한 선택입니다.

Datadog RUM 대비 예상 절감: DAU 5만 기준 Datadog ~$1,500~3,000/월 → A안 ~$200~500/월 (70~85% 절감)
